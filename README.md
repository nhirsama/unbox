# unbox

一个纯前端的国籍/地区归因与伪装一致性探针。它不是计算“一般隐私泄露了多少”，而是汇总网页可见且具有地理差异的环境线索，估算外部观察者会把当前浏览器/网络环境归因到哪个国家或地区，以及这些线索是否像经过代理、跨区配置或环境伪装。

项目按 **HTTPS 静态站** 设计，不需要自有后端；出口 IP、GeoIP、TLS 回显和连通性目标均使用 HTTPS。

## 已实现模块

- 语言与地区环境：`navigator.language`、`navigator.languages`、`Intl`、时区，用于判断语言/地区归因。
- 日历系统归因：检测 `Intl.DateTimeFormat` 是否默认或显式支持农历、民国纪年、日本年号、佛历等地区化 Calendar Systems。
- 区域化 Emoji 渲染：检测旗帜 Emoji，尤其是特殊地区旗帜在不同发行地区/系统字体策略下的渲染差异。
- 字体地区归因：仅探测简体中文/繁体中文地区常见字体，不导出通用字体清单或文字渲染 hash。
- 键盘布局归因：使用 Keyboard Layout Map 识别 AZERTY、QWERTZ、JIS 等具有地区差异的键盘布局。
- 语音包地区归因：读取 SpeechSynthesis 可见语音包语言标签，本地 TTS 和默认语音可反映系统语言/地区环境。
- TLS/浏览器栈归因：通过公开 HTTPS TLS 回显端点观察 JA3/JA4 摘要、国密套件迹象和国内浏览器栈特征。
- 出口 IP 与 GeoIP：`ipify`、`ipapi.co` 等公开 HTTPS 接口，观察第三方看到的出口 IP 与国家/地区。
- WebRTC 出口一致性：收集 ICE 候选；扩展模式使用公共 STUN，并通过 HTTPS GeoIP 备选查询候选公网 IP 的地区归属。
- 连通性矩阵：HTTPS 图片资源加载探测，包含大陆常用站点、全球社交/通讯站点、常见被屏蔽站点，以及港台日韩俄伊、新加坡、马来西亚、泰国、越南等强地域站点；只有 `Image.onload` 才计为可达，404/403/HTML 错误页/解码失败/阻断/超时都计为失败，部分站点配置多个图片候选 URL，任一候选成功才计为该站点可达。结果会按站点组统计成功率、失败率、中位耗时和相对基线速度，生成各地区网络环境分与代理/分流异常分。
- 地域站点缓存/DNS 预热：扩展模式下对大陆强地域站点图片资源做两次加载计时，寻找 DNS/连接/缓存预热迹象；只有真实图片加载成功才参与预热判断。

已移除与国籍/地理归因弱相关的泛设备识别模块：UA/UA-CH 泛枚举、硬件并发、内存档位、屏幕、Canvas/WebGL/Audio/WebGPU、存储、权限状态、电池、媒体设备、Gamepad、USB/Bluetooth、端口扫描等。

## 输出结果

- **国籍/地区归因候选**：外部观察者可能认为你属于哪个国家/地区，以及对应置信度。
- **连通性矩阵评分**：把站点探测结果聚合为中国大陆、繁中地区、香港、日本、韩国、俄罗斯、伊朗、新加坡、马来西亚、泰国、越南等网络环境分；同时给出被屏蔽目标失败率、全球社交可达率、强地域站点耗时等统计依据。
- **伪装/不一致概率**：出口 IP、WebRTC/STUN、TLS/浏览器栈、语言、时区、日历、字体、键盘、语音包等信号相互冲突时，给出代理/跨区/伪装概率。
- **归因证据**：每条证据的来源、指向国家/地区和置信度。
- **一致性信号**：只保留会影响国籍/地区归因或伪装判断的信息。

## 构建与部署

```bash
pnpm build
pnpm preview
```

`pnpm build` 生成 `dist/`，可直接部署到任意 HTTPS 静态站平台，例如 Cloudflare Pages、Vercel、Netlify、GitHub Pages 或 Nginx/Caddy 静态目录。

## HTTPS 备选策略

- 所有内置 IP API、GeoIP 查询、TLS 回显和连通性矩阵目标均优先使用 HTTPS。
- WebRTC 的 host-only 模式不依赖外部服务；扩展模式会使用公共 STUN 发现公网候选，再用 HTTPS GeoIP 接口做地区备选归因。
- 如果某个公开 IP/TLS/GeoIP 接口不可用，前端会继续使用其他公开接口和本地浏览器信号形成归因。
- 第三方连通性矩阵使用图片资源加载事件判断“真实资源是否可加载”；前端仍不读取响应 Header 或 Body。
- 出口 IP 被描述为“第三方服务观察到的出口 IP”，不直接等同于物理所在地或个人身份。

## 连通性评分逻辑

- 每个站点组先统计 `total / ok / failed / okRate / failRate / medianMs`。
- 矩阵分为四类：`baseline` 基线、`mainland-service` 大陆常用服务、`restriction-signature` 限制/阻断特征、`regional-service` 地区强站点。
- `global-reference` 只作为耗时基线和普通连通性参考，不直接等同于某个国家/地区归因。
- 中国大陆网络环境分主要来自：大陆常用站点可达率、地缘政治敏感站点失败率、全球社交/通讯站点失败率，以及“大陆站点可达 + 敏感目标失败”的组合特征；单纯大陆常用站点可达只能形成很弱的网络线索。
- 其他地区网络环境分主要来自：对应强地域站点可达率和相对低耗时；这些是网络环境证据，权重低于出口 IP、语言/时区等强归因信号。
- 强地域站点使用硬门槛：对应地区站点组可达率低于 90% 时，不生成该地区网络归因；例如 75% 只表示部分可达，不足以认为网络环境属于该地区。
- 代理/分流异常分主要来自：出口 IP 国家/地区与连通性矩阵指向不一致、IPv4/IPv6 或多个 IP API 出口不一致、CN IP 下敏感/社交站点大量可达、非 CN IP 下仍呈现大陆阻断特征等。
- 国籍/地区候选采用分层权重：出口 IP、WebRTC/STUN、时区和主语言权重最高；连通性矩阵是中等网络证据；字体、语音包、键盘、日历和 Emoji 多数是本地环境增强证据，主要用于一致性判断，不会单独压过强网络/地区信号。

## 归因融合逻辑

- 不再线性堆分。每条证据先按来源封顶，再按证据簇封顶，避免同一类信号无限叠加。
- 证据簇分为：
  - `network`：出口 IP、WebRTC/STUN、连通性矩阵。
  - `localization`：主语言、备用语言、时区、Intl 日历。
  - `system`：字体、语音包、键盘、TLS/浏览器栈、Emoji、地域缓存。
- 同一国家/地区如果同时获得 `network + localization` 支持，会得到一致性加成；只有本地弱信号一致不会给很高置信度。
- 伪装/不一致概率不再由“第二名国家有分”直接推高，而是按高权重来源之间的跨簇冲突计算：例如 IP 与时区/语言冲突、HTTPS 出口与 WebRTC/STUN 冲突、出口 IP 与连通性矩阵模式冲突。
